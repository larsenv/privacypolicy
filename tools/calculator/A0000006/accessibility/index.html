Accessibility

Accessibility information for the current iPhone and iPad release.

This page provides concrete public information about the app’s accessibility posture without overstating features that should be validated on real devices before submission labels are claimed. Repo validation alone is not enough; app-specific tasks to validate include VoiceOver navigation, larger text, dark interface contrast, compact iPhone, large iPhone, 11-inch iPad, 13-inch iPad, portrait and landscape layouts, and reachable support links.

Current accessibility focus

Readable, labeled, adaptive calculator workflows.

Accessibility claims should be based on common-task testing across every supported device class and the accessibility features people rely on.

Labeled controls

The app is designed around labeled controls and structured grouping across calculator, help, privacy, support, and history surfaces.

Scalable typography

The interface is designed to support larger text across top bars, input/output areas, settings, and overlays, with layout validation required before making public labels.

Appearance support

The current release supports light and dark appearance modes and aims for strong contrast and readability.

Device coverage

Testing should include compact iPhone, standard iPhone, large iPhone, 11-inch iPad, and 13-inch iPad configurations.

Validation checklist

Common tasks to test before App Store accessibility claims.

1. App shell

Launch the app, open the menu, switch calculator types, open settings, open help, and reach privacy/support pages without getting trapped.

2. Calculator workflows

Enter values, move cursors, clear entries, calculate results, switch angle modes, and review history in Basic, Scientific, Percentages, Fractions, Graphing, Money, and Time.

3. Appearance and size

Test light mode, dark mode, standard text, larger text, portrait, landscape, compact iPhone, large iPhone, and iPad sizes.

4. Report issues precisely

Send device model, iOS/iPadOS version, orientation, calculator type, accessibility feature, expected behavior, actual behavior, and screenshots when useful.
