What to include

Support requests move faster when they are specific

Basic / scientific

Include the exact expression, the expected result, and the result you received. For equation workflows, include the exact equation you entered.

Graphing

Include the function text, the viewport or range you were using, and whether the issue involves tracing, intersections, or roots.

Fractions / percentages

Include the exact values entered in each field and the operator or chart mode in use.

Money / time / unit conversion

Include the source and destination units or currencies, the date/time inputs if relevant, and whether the device had network access.

Quick troubleshooting

Common things to check first

Committed history

Tap = to commit a result before expecting it to appear in saved history.

Settings changes

If a preference seems stuck, relaunch the app after changing the setting and confirm the active calculator mode.

Money conversions

If a live rate is required, confirm that the device has network access and retry the conversion.

Precise reproduction

For calculation issues, the exact input sequence matters. Include it in the report whenever possible.
