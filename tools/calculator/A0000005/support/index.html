Accessibility feedback
You can use the same support email, or the accessibility page, for accessibility-specific reports.

Before sending a report, include the device model, iOS / iPadOS version, what happened, what you expected to happen, and whether the issue appeared in card, grid, timeline, or composer screens.

Quick troubleshooting

  • Confirm whether the countdown is stored as date-only or date-and-time.
  • If a time-sensitive result looks wrong, include the device’s current time zone and whether the clock or time zone changed recently.
  • If a custom background photo is missing, verify that the event still points to the intended photo and that photo access was not revoked mid-flow.
  • If a history-pack event looks out of place, mention which history pack is enabled and the event title involved.
  • For layout issues, attach a screenshot and include the device orientation, text size, and appearance mode if relevant.

Privacy-related requests

  • Review the Privacy Policy for current-release data-handling details.
  • Use the Privacy Choices page for direct local-data management guidance.
  • If you need to ask a privacy question, send it to the support email above.
