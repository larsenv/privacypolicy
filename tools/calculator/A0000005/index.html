At a glance

Core countdowns, notes, view preferences, onboarding and EULA flags, enabled history packs, optional local background photos, and a local recovery copy of the event store stay inside the app’s private sandbox so the experience remains fast and consistent between launches.

What makes this site app-specific

Local-first countdown dataThe policy is specific to Project Countdown’s storage of event titles, notes, dates, precision modes, view preferences, enabled history packs, and local backup files.
Optional photo workflowThe site documents the photo-library path accurately: access is requested only when you choose a custom background photo, and selected images are stored locally in the app sandbox.
Time-sensitive behaviorThis release stores both date-only and date-and-time events and refreshes countdown presentation when the device’s time, time zone, locale, or calendar day changes.
Apple Ads attribution stays offThe codebase contains a default-off Apple Ads attribution path, but the current release keeps it disabled and does not request attribution.
