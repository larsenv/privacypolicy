Accessibility

Accessibility information for Project Countdown.

Important note

Accessibility quality should always be verified on supported devices, text sizes, OS versions, and accessibility settings in real-world use.

Current accessibility posture

Control clarityImportant controls and menu surfaces are designed to be visually distinct and clearly labeled across the main countdown, grid, timeline, and composer flows.
Readable sizingThe current release uses scalable typography and adaptive layout behavior so larger text sizes can expand or scroll instead of clipping key content.
Appearance supportThe app supports light and dark appearances to help preserve contrast across common usage environments.
Touch-first interactionPrimary controls are designed around generous touch targets and structured grouping to reduce ambiguity in sheets, menus, and event-management flows.

Best way to report an accessibility issue

  1. Describe the device model and iOS / iPadOS version.
  2. Identify the screen or flow where the issue occurred, such as card view, grid view, timeline view, or the event composer.
  3. List the accessibility feature in use, such as VoiceOver, Larger Text, Bold Text, Reduce Motion, or increased contrast.
  4. Describe the exact problem and the expected behavior.
  5. If possible, include a screenshot or screen recording and note whether the issue happens every time.
