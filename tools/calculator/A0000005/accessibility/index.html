Accessibility

Accessibility information for Project Countdown.

Project Countdown is intended to be usable on iPhone and iPad with clear labels, readable layout, touch-friendly controls, light and dark appearance support, and direct accessibility feedback.

Email Accessibility FeedbackSupport

App accessibility posture

The app uses structured controls, grouped sheets, readable typography, and adaptive layout behavior for card, grid, timeline, and composer flows.

Appearance and motion

The current code path supports light and dark appearances and includes reduced-motion considerations in the broader QA checklist.

Website accessibility

This website uses semantic headings, visible links, skip navigation, keyboard focus styles, responsive layouts, and plain-language support paths.

Report an issue

Email trentshurts@gmail.com with the device, OS version, screen, accessibility setting, expected behavior, and actual behavior.

Recommended validation checklist

Assistive technology

  • VoiceOver navigation through event cards, timeline controls, and composer sheets.
  • Voice Control or switch-style activation for primary actions.

Layout

  • Small iPhone, large iPhone, iPad portrait, and iPad landscape.
  • Larger Text, Bold Text, Reduce Motion, and high-contrast settings.
